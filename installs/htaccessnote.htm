.htaccess File Notes

If your .htaccess file is not working properly, it will more than likely be due to one of the following reasons:

  1. The .htaccess file does not exist. Please check that the file exists; you may still need to rename the file from "htaccess.txt" file to ".htaccess".
  2. If you are installing Tradingeye in a subfolder or subdomain, please ensure you edit your .htaccess file accordingly. See the Developer Manual for more information.
  3. Linux hosting: The web server should allow the use of the .htaccess Allowoverride All setting. You may wish to check this feature with your hosting provider.
  4. Windows hosting: IISRewrite is used for safe URLs. We are using a trial version which has limitations - the evaluation version has all the functionality of the full version except that it will stop performing URL rewrites after two hundred manipulations, and will only begin to perform URL rewrites again after the web server is restarted.
